## Cara Berkontribusi

### 🐛 Melaporkan Bug
Nemu *error* atau sesuatu yang janggal?
- Buka [Issue](https://github.com/liwirya/cellyn-whatsapp-bot/issues).
- Jelaskan langkah-langkah untuk mereproduksi bug tersebut, apa ekspektasimu, dan apa yang sebenarnya terjadi.
- Sertakan *screenshot* atau log kalau ada — ini bakal ngebantu banget!

### 💡 Mengusulkan Fitur
Punya ide keren buat bikin Cellyn makin jago?
- Buka Issue baru dan beri label `enhancement`.
- Ceritakan kegunaan fitur tersebut dan usulan solusinya seperti apa.

### 🛠️ Pull Request (PR)
Siap nulis kode? Mantap! Berikut alurnya:
1. **Fork** repositori ini.
2. Buat *branch* baru untuk fiturmu: `git checkout -b feature/nama-fitur`.
3. Buat **commit** dengan pesan yang jelas dan deskriptif.
4. **Push** ke fork kamu, lalu buka Pull Request.
5. Tunggu sebentar! Kami akan me-review kodemu dan merespons secepat mungkin.

## Gaya Penulisan Kode (Code Style)

Biar kode kita tetap rapi dan konsisten:
- Gunakan **ES Modules** (`import/export`).
- Ikuti struktur plugin yang sudah ada supaya seragam.
- Tambahkan komentar **JSDoc** untuk fungsi-fungsi yang cukup rumit.
- Pastikan plugin tetap fokus — cukup satu fungsi/tanggung jawab per *file*.

Hey there! 👋 First of all, thank you so much for wanting to contribute to Cellyn. We really appreciate your time and effort! 🎉

A quick note on our project's roots: **Cellyn** is built by [Liwirya](https://github.com/liwirya). The architecture and core structure of this bot are inspired by and cloned from [Katsumi](https://github.com/nat9h/Katsumi) by [nat9h](https://github.com/nat9h) — massive shoutout to them for laying down such a solid foundation!

## How You Can Help

### 🐛 Squashing Bugs
Found a glitch? We’d love to fix it.
- Open an [Issue](https://github.com/liwirya/cellyn-whatsapp-bot/issues).
- Tell us exactly how to reproduce it, what you expected to happen, and what actually happened.
- Drop in some screenshots or logs if you can—they help a ton!

### 💡 Suggesting Features
Got a cool idea to make Cellyn better?
- Open an Issue and tag it with the `enhancement` label.
- Tell us the use case and how you think the solution should work.

### 🛠️ Pull Requests
Ready to write some code? Awesome! Here’s the workflow:
1. **Fork** the repo.
2. Create a new branch for your feature: `git checkout -b feature/your-feature-name`.
3. **Commit** your changes with clear, descriptive messages.
4. **Push** to your fork and open a Pull Request (PR).
5. Hang tight! We'll review it and get back to you as soon as possible.

## Code Style

Let's keep the codebase clean and consistent:
- Use **ES Modules** (`import/export`).
- Stick to the existing plugin structure so everything feels at home.
- Add **JSDoc** comments for any complex functions to help out future devs.
- Keep your plugins focused — one specific responsibility per file.
